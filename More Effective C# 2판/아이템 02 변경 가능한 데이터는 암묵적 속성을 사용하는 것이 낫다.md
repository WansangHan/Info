# 암묵적 속성의 특성
* 암묵적 속성을 이용하면 컴파일러는 속성 값을 저장할 뒷단 필드를 추가하고, 임의로 이름을 부여한다. 이 부분 말고는 데이터 멤버와 코드상으로 똑같이 동작한다. 성능까지도.
* 암묵적 속성은 명시적 속성과 동일한 접근 한정자를 지원한다. 필요하다면 set에만 접근 권한을 제한할 수 있다.
* 암묵적 속성의 장점은 개발자의 생산성을 높이고 클래스의 가독성을 높이는 데 있다.
* 암묵적 속성도 virtual, interface등에 사용할 수 있다.
* 암묵적 속성을 virtual로 선언한 경우라 하더라도 파생 클래스에서 컴파일러가 생성한 뒷단 필드에 직접 접근할 수 없다.

# 암묵적 속성의 장점
* 속성 접근자를 이용하여 외부에서 접근 가능한 데이터를 추가하려는 경우, 암묵적 속성을 이용하면 코드를 읽기 쉽다.
* 암묵적으로 속성을 선언하면 실제 의도를 모호하게 만드는 추가 정보로 코드가 지저분해지는 일을 막아주어, 코드를 읽는 사람 모두가 작성자의 의도를 쉽게 파악할 수 있다.
* 암묵적 속성을 사용하면 명시적 속성으로 구현부를 추가해도 클래스의 바이너리 호환성이 유지된다.
* 암묵적 속성을 사용하면 여전히 속성을 사용하고 있으므로 데이터 검증 코드를 한 군데만 두면 된다.

# 암묵적 속성의 제약
* Serializeable 특성을 사용한 타입에는 사용할 수 없다. 
    * 클래스를 위한 영구 파일 저장소의 포멧은 암묵적 속성을 위해 컴파일러가 자동 생성한 뒨단 필드의 이름에 영향을 받는다. 이 이름이 변경되지 않을 것임을 보장할 수 없다.
