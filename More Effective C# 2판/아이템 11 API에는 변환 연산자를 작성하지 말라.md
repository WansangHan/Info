## 변환 연산자
* 변환 연산자는 클래스 간 대체 가능성을 지원하기 위한 기능이다.
    * 대체 가능성이란 하나의 클래스를 다른 클래스로 대체할 수 있음을 말한다.
* 변환 연산자를 정의한다는 것은 다른 타입으로 대체할 수 있음을 컴파일러에게 알려주는 것이다.

## 변환 연산자의 문제점
* 종종 대상 타입으로 완벽하게 변환을 수행하지 못하는 경우가 발생하면 미묘한 오류를 일으킨다.
    * 대상 타입으로 변환된 객체의 상태를 변경하는 것과 이전 객체의 상태를 변환하는 것의 결과가 서로 다를 수 있다.
* 변환 연산자가 임시 객체를 반환하여, 이 객체의 상태를 변환하는 동작을 하더라도 아무런 영향을 미치지 못하고 결국 가비지 콜렉터에 의해서 제거되어 버리는 경우도 있을 수 있다.
* 변환 연산자를 호출하는 규칙은 객체의 런타임이 아니라 컴파일 타임 타입을 기준으로 한다.
    * 필요한 변환 연산자를 호출하기 위해 캐스트를 여러 번 수행해야 하는 경우가 생기곤 하는데, 이는 코드의 유지 관리를 더 어렵게 만든다.

## 변환 연산자의 문제점에 대한 대응
* 생성자
    * 생성자는 새로운 객체를 만든다는 사실을 명확히 알려준다.
    * 변환 연산자는 오류 지점을 찾기 어렵게 만든다.
        * 생성자 내에서 객체를 새로 만들기 때문에 기존 객체에 변경점이 없고, 새로 만들어진 객체를 실수로 버릴 수 있다.
