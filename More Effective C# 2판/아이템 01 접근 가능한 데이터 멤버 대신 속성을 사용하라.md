## 속성의 특성
* 속성은 Getter와 Setter의 접근 제한을 다르게 설정할 수 있다.
* 자동 속성을 사용하면 데이터 멤버를 명시적으로 선언할 필요가 없다.
* 속성은 데이터 멤버를 Public으로 노출하면서도 객체 지향에 필요한 캡슐화를 유지할 수 있다.
* 속성은 데이터 멤버처럼 접근할 수 있지만 실제로는 메서드로 구현된 언어 요소이다.
* .Net 프레임워크는 public 데이터 멤버 대신 속성을 사용할 것이라고 가정한다.
    * .Net 프레임워크의 데이터 바인딩 클래스들은 Public 데이터 필드가 아니라 속성에 대해서만 동작한다.
* 속성은 virtual로도 설정할 수 있다.
* 속성은 인터페이스를 정의할 때도 사용할 수 있다.
* 어떤 타입이 여러 개의 요소를 포함하는 경우라면 인덱서라는 속성을 사용할 수 있다.
    * 인덱서는 이름을 가질 수 없다. 따라서 여러 개의 인덱서를 선언하려면 각기 매개변수 목록이 달라야 한다.
    * 다차원 인덱서도 만들 수 있다.
* 데이터 멤버보다 속성이 조금 느리지만 체감하지 못할 정도이며 속성이 인라인화 된다면 같은 속도이다.
* 암묵적 속성 문법을 사용하면 코드를 크게 줄일 수 있다. get, set을 자동 생성 해준다.

## 속성의 장점
* 코드를 수정해야할 때 Public 속성으로 선언했다면, 한 곳만 수정하면 된다.
    * 코드를 수정해야할 때 Public 데이터 멤버로 선언했다면, 변수에 값을 설정하는 코드를 모두 찾아 수정해야한다.
* 속성은 멀티스레드도 쉽게 지원할 수 있다. get, set에 동기화 기능을 구현하면 된다.
* 속성은 메서드 속성이기 때문에 ref, out 키워드를 통해 외부로 데이터가 노출될 수 없다.
* 데이터 멤버와 속성은 소스 수준에서는 호환성이 있지만 바이너리 수준에서는 전혀 호환성이 없다. 수정한다면 관련 코드가 전부 재 컴파일 된다.
