## 참조 반환의 문제점
* 참조 타입을 반환하는 모든 멤버는 해당 객체의 핸들을 반환한다. 이는 호출측에 객체의 내부 구조를 가리키는 핸들을 넘겨주므로, 이 객체를 반환한 타입을 거치지 않고도 참조를 통해 객체의 상태를 수정할 수 있다.

## 내부 객체를 보호하기 위한 전략
* 값 타입
    * 복사본을 넘긴다.
    * 클라이언트가 복사본을 변경해도 객체의 내부 상태에는 영향이 미치지않는다.
* 변경 불가능한 타입
    * 반환된 값을 수정할 수 없기 때문에 안전하다.
* 인터페이스
    * 클래스의 기능을 제한적으로 정의한 인터페이스를 공개하면 의도하지 않게 내부 데이터가 변경되는 것을 막을 수 있다.
* 클라이언트에게 내부 데이터를 수정하도록 허용하되, 수정된 내용에 대해서 반드시 검증을 수행하도록 하여 이러한 기법을 일반화할 수도 있을 것이다.
* 래퍼
    * 내부 객체로의 접근을 최소화 하도록 래퍼 객체를 만들고, 그 래퍼의 인스턴스를 건네는 방법
